Для того чтобы свести к минимуму вероятность потери данных в результате сбоев в работе оборудования или неосторожных действий пользователя, система UMI.CMS предоставляет модуль резервного копирования.

Под "событием" понимается изменение данных страницы/объекта сайта при его сохранении.

Кнопка Вести резервное копирование по умолчанию включена. Также вы можете задать максимальное количество копий для каждой отдельной страницы/объекта сайта и максимальное время хранения каждой копии объекта на сервере (в целях экономии дискового пространства).

В этом случае при каждом сохранении любой из страниц вашего сайта будет создаваться новая копия страницы, которую можно восстановить. В редакторе содержимого (страницы сайта, новости) есть область История изменений и версии страниц, где вы можете восстановить одну из ранее сохранённых версий.
